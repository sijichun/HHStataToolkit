# 决策树、随机森林的Stata插件

现在我想要写一个决策树、随机森林的C实现，供Stata使用。目前的设计想法包括：

* 首先实现从Stata中调数据到C的接口，统一将数据转为单精度float（也许能加快速度？）
* 对于每一个变量，实现一个计算十分位数的函数（其输入为指针数组，计算对象为指针数组所指向的具体数字）。9个十分位数可能会重复，所以还要实现去重的功能。比如十分位数为（1,1,1,2,2,4,6,8,10），实际使用（1,2,4,6,8,10）即可
* 遍历每个变量的十分位数，区分出left和right两个节点，计算impurity，并找到能使得impurity下降最快的方法，进行区分
* 维护一个数组，用于保存每个观测所属的节点。其编码可以使用二进制的位运算符。比如，根节点初始化为0，从根节点出发，左边的标记为0，右边的标记为1；从左边节点出发，左边的标记为0，右边的标记为1；从右边的节点出发，左边的标记为10，右边的标记为11（其中数字均使用二进制表示）。
* 额外维护一个completed数组，初始化为空。一旦某个节点无法继续划分（达到最大深度、杂度无法下降）则计入completed数组。
* 每一轮*并行的*对各个节点进行计算，寻找最好的划分方法。比如，如果现在的节点为（0,1,10,11），就并行的对四个节点探索所有的变量、分位数的划分方法，找到杂度下降最快的方法，并记录杂度下降幅度
* 找到下降最快的那个，比如11下降最快，将其划分为（11,111）两个模块，如此现在有（0，1，10，11，111）这几个模块，再并行探索（11,111）两个模块的杂度下降，并跟(0,1,10)的杂度下降幅度进行比较，找到最快的，以此类推
* 每一步都判断（最大层数、最大叶子节点数）是否达到。如果达到则停止，否则继续。
* 不同目的的杂度函数计算可能不同，所以需要用函数指针的形式实现动态调整杂度计算。
* 还需要实现target的划分功能，即使用一部分数据构建树，在另一部分数据上做预测。
* 最终只要返回给stata这个所属节点数组即可（包括target=0/1）。
